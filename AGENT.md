# AGENT.md для `ScreenUI`

Этот файл задает обязательные правила работы агента в репозитории `ScreenUI`.

## 1. Главное обязательное правило

Это правило обязательно для каждой новой сессии и для любой правки без исключений.

- Никогда не ломать русские символы, кириллицу и кодировку в существующих файлах.
- Все комментарии в коде писать только на русском языке.
- Все новые и обновляемые `README.md` писать только на русском языке.
- При редактировании текстовых файлов использовать UTF-8 и проверять, что после сохранения текст читается в IDE без ручной перекодировки.
- Если файл уже в другой кодировке, нельзя молча перекодировать его вместе с функциональными правками. Смена кодировки должна быть осознанной и проверяемой.
- Запрещено оставлять mojibake вида `Р...`, `С...`, `Ð...`, `Ñ...` и другие следы битой перекодировки.

## 2. Роль репозитория

`ScreenUI` — UI/design package.

Допустимое содержимое:
- EEZ project;
- generated shared/frontend meta;
- concrete LVGL/EEZ adapter;
- tooling генерации;
- vendor LVGL.

Недопустимое содержимое:
- host/business runtime;
- protocol runtime;
- backend-логика.
